# Home Manager File Management Guide

## 📁 File Management Methods in Home Manager

### 1. `lib.homeManager.mkStoreOutOfSymlink` (Recommended for most cases)

**Best for:**
- Configuration files that might be modified by applications
- Files that applications need to write to
- Large files that shouldn't be copied to the store
- Files with special permissions

**Usage:**
```nix
home.file.".config/app/config.json" = lib.homeManager.mkStoreOutOfSymlink {
  source = ./path/to/source.json;
  executable = true;  # Optional: for executable files
};
```

**Examples in your config:**
- `~/.config/opencode/opencode.json` - OpenCode configuration
- `~/.config/sketchybar/sketchybarrc` - Sketchybar configuration
- `~/.aerospace.toml` - Aerospace configuration
- `~/.config/sketchybar/workspace_update.sh` - Executable script

### 2. `.text` attribute (For simple text files)

**Best for:**
- Small configuration files that never change
- Files generated entirely by Nix
- Simple JSON/YAML/TOML files

**Usage:**
```nix
home.file.".config/app/config.json".text = ''
  {
    "key": "value"
  }
'';
```

**Examples in your config:**
- `~/.config/lm-studio/config.json` - Simple JSON config

### 3. `.source` attribute (Legacy method)

**Best for:**
- Simple cases where you don't need special handling
- Files that are truly read-only

**Usage:**
```nix
home.file.".config/app/config.json".source = ./path/to/source.json;
```

## 🎯 When to Use `mkStoreOutOfSymlink`

### ✅ Use `mkStoreOutOfSymlink` when:

1. **Application modifies the file** - Apps like sketchybar, aerospace, etc. that might update their own configs
2. **Executable scripts** - Any shell scripts or binaries that need execute permissions
3. **Large files** - Files >1MB that shouldn't be copied to the Nix store
4. **Permission-sensitive files** - Files that need specific permissions (644, 755, etc.)
5. **Symlink-friendly apps** - Applications that work better with symlinks than copies

### ❌ Avoid `mkStoreOutOfSymlink` when:

1. **File must be in the Nix store** - For files that need to be part of the Nix closure
2. **File is generated dynamically** - Use `.text` for files created entirely by Nix
3. **Simple read-only configs** - Small JSON/YAML files that never change

## 🔧 Best Practices

### 1. **For Application Configs**
```nix
# Good: Application configs that might be updated
home.file.".config/sketchybar/sketchybarrc" = lib.homeManager.mkStoreOutOfSymlink {
  source = ./config/sketchybar/sketchybarrc;
  executable = true;
};
```

### 2. **For Generated Configs**
```nix
# Good: Simple configs generated by Nix
home.file.".config/lm-studio/config.json".text = lib.mkForce ''
  {
    "bootstrappedByHomeManager": true
  }
'';
```

### 3. **For Executable Scripts**
```nix
# Good: Scripts that need to be executable
home.file.".config/sketchybar/workspace_update.sh" = lib.homeManager.mkStoreOutOfSymlink {
  source = ./config/sketchybar/workspace_update.sh;
  executable = true;
};
```

## 📋 Migration Guide

### From `.source` to `mkStoreOutOfSymlink`:

**Before:**
```nix
home.file.".config/app/config.json".source = ./config.json;
```

**After:**
```nix
home.file.".config/app/config.json" = lib.homeManager.mkStoreOutOfSymlink {
  source = ./config.json;
};
```

### Adding Executable Permission:

**Before:**
```nix
home.file.".script.sh".source = ./script.sh;
```

**After:**
```nix
home.file.".script.sh" = lib.homeManager.mkStoreOutOfSymlink {
  source = ./script.sh;
  executable = true;
};
```

## 🚀 Performance Considerations

- **Symlinks are faster** - No copying of large files
- **Atomic updates** - Changes are atomic (either fully there or not)
- **Store efficiency** - Doesn't bloat the Nix store with file copies
- **Rollback friendly** - Easy to rollback to previous versions

## 🎓 Advanced Usage

### With Custom Permissions:
```nix
home.file.".config/app/data.db" = lib.homeManager.mkStoreOutOfSymlink {
  source = ./data.db;
  permissions = "600";  # Read/write for owner only
};
```

### With Post-Install Hooks:
```nix
home.file.".config/app/config.json" = lib.homeManager.mkStoreOutOfSymlink {
  source = ./config.json;
  postInstall = ''
    echo "Config installed at $out"
    # Additional setup commands
  '';
};
```
